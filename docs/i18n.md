# Internationalization (i18n)

This document explains the internationalization setup and how to manage multiple languages in the minimal-cv project.

## Overview

The project uses [next-intl](https://next-intl-docs.vercel.app/) for internationalization, supporting:

- **English** (`en`) - Default locale
- **Chinese Simplified** (`zh`) - Primary locale  
- **Japanese** (`ja`)

## Configuration

### Core Setup

The i18n configuration is defined in `i18n.ts`:

```typescript
export const locales = ['en', 'zh', 'ja'] as const;
export const defaultLocale = 'zh' as const;
export type Locale = (typeof locales)[number];

export const localeLabels: Record<Locale, string> = {
  en: 'English',
  zh: '中文',
  ja: '日本語'
};
```

### Middleware

The root `middleware.ts` handles locale routing:

```typescript
export default createMiddleware({
  locales,
  defaultLocale,
  localePrefix: 'always', // Always show locale in URL
  localeDetection: true,  // Auto-detect user's preferred language
  alternateLinks: true    // Generate alternate links for SEO
});
```

### App Router Structure

The app uses Next.js App Router with dynamic locale segments:

```
app/
└── [locale]/
    ├── layout.tsx    # Locale-specific layout
    └── page.tsx      # Main CV page
```

## Locale Detection

The system detects the user's preferred locale through:

1. **URL Path**: `/en/`, `/zh/`, `/ja/`
2. **Browser Language**: `Accept-Language` header
3. **Fallback**: Default locale (`zh`)

### Locale Utilities

Helper functions in `lib/i18n-utils.ts`:

```typescript
// Validate locale
isValidLocale(locale: string): locale is Locale

// Extract locale from pathname
getLocaleFromPathname(pathname: string): Locale | null

// Create localized paths
createLocalizedPath(path: string, locale: Locale): string

// Get text direction for RTL support
getDirection(locale: Locale): 'ltr' | 'rtl'
```

## Using Translations

### In Components

```typescript
import { useTranslations } from 'next-intl';

function MyComponent() {
  const t = useTranslations('section');
  
  return (
    <h2>{t('education')}</h2>
  );
}
```

### Server Components

```typescript
import { getTranslations } from 'next-intl/server';

async function MyServerComponent({ locale }: { locale: string }) {
  const t = await getTranslations({ locale, namespace: 'section' });
  
  return <h2>{t('education')}</h2>;
}
```

### Type-Safe Translations

Use the `NavigationMessages` type for type safety:

```typescript
import { type NavigationMessages } from '@/lib/i18n-utils';

const t = useTranslations() as (key: keyof NavigationMessages) => string;
```

## CV Data Localization

### Data Structure

Each locale has its own CV data file:

```
data/
├── cv.toml       # Default CV data
├── cv.en.toml    # English CV data
├── cv.zh.toml    # Chinese CV data
└── cv.ja.toml    # Japanese CV data
```

### Loading Localized Data

The `getCVData` function automatically loads the correct data:

```typescript
import { getCVData } from '@/lib/load-cv-data';

// Loads cv.zh.toml by default
const data = await getCVData();

// Loads specific locale
const englishData = await getCVData('en');
```

### Fallback Strategy

If a locale-specific file is missing:

1. Falls back to default locale (`zh`)
2. Returns sample data as last resort
3. Logs warnings for debugging

## Locale Switcher

The `LocaleSwitcher` component provides language switching:

### Features

- **Visual Feedback**: Active locale highlighting
- **Smooth Transitions**: Using React transitions
- **Accessibility**: ARIA labels and keyboard navigation
- **Progressive Enhancement**: Works without JavaScript
- **Debug Mode**: Shows current locale in development

### Implementation

```typescript
function LocaleSwitcher() {
  const pathname = usePathname();
  const currentLocale = getLocaleFromPathname(pathname) || defaultLocale;
  const [isPending, startTransition] = useTransition();

  const handleLocaleChange = (locale: Locale) => {
    startTransition(() => {
      const href = createLocalizedPath(pathname, locale);
      router.push(href);
    });
  };
  
  // Render locale options...
}
```

## SEO and Metadata

### Localized Metadata

Each locale has specific metadata in `app/[locale]/layout.tsx`:

```typescript
export function generateMetadata({ params }: { params: { locale: string } }) {
  const locale = params.locale as Locale;
  
  const titles = {
    en: "Keyou Zheng - CV | Graduate Student & Researcher",
    zh: "郑恪悠 - 简历 | 研究生 & 研究员", 
    ja: "郑恪悠 - 履歴書 | 大学院生 & 研究者"
  };
  
  return {
    title: titles[locale],
    alternates: {
      languages: Object.fromEntries(
        locales.map((loc) => [loc, `/${loc}`])
      ),
    },
    // OpenGraph, Twitter cards...
  };
}
```

export const localeLabels: Record<Locale, string> = {

  ja: '日本語',

### 3. Create CV Data

### 4. Update Metadata

Add French metadata in `app/[locale]/layout.tsx`.

### 5. Test and Validate

- Check SEO alternate links

## RTL Language Support

For right-to-left languages:

### 1. Update Direction Helper

```typescript
export function getDirection(locale: Locale): Direction {
  const rtlLocales: Locale[] = ['ar', 'he']; // Add RTL locales
  return rtlLocales.includes(locale) ? 'rtl' : 'ltr';
}
```

### 2. Apply Direction in Layout

```typescript
const direction = getDirection(locale as Locale);

return (
  <html lang={locale} dir={direction}>
    {/* ... */}
  </html>
);
```

### 3. RTL-Aware Styling

Use Tailwind CSS RTL utilities:

```css
.rtl:text-right  /* Right align in RTL */
.rtl:ml-4        /* Margin left becomes margin right */
```

## Best Practices

### Translation Keys

- Use nested namespaces: `section.education`
- Keep keys descriptive: `label.supervisor`
- Avoid overly specific keys: `button.submit` vs `button.submitForm`

### Content Guidelines

- **Consistency**: Use same terminology across locales
- **Context**: Provide context for translators
- **Pluralization**: Handle singular/plural forms appropriately
- **Date Formats**: Use locale-appropriate formats

### Performance

- Translation files are loaded per locale
- Dynamic imports reduce bundle size
- Static generation for better performance

### Debugging

Enable debug mode in development:

```typescript
// Shows current locale and path
{process.env.NODE_ENV === 'development' && (
  <div className="debug-info">
    Current: {currentLocale} | Path: {pathname}
  </div>
)}
```

## Common Issues

### Locale Not Detected

- Check middleware configuration
- Verify locale in URL path
- Check browser Accept-Language header

### Translation Missing

- Verify key exists in message file
- Check namespace usage
- Use fallback translations

### Routing Issues

- Ensure middleware matcher includes paths
- Check createLocalizedPath function
- Verify locale validation

### Build Errors

- Validate all translation files
- Check CV data structure consistency
- Ensure all locales have required files
