# Internationalization

This project uses `next-intl` with App Router locale segments under `app/(cv)/[locale]` and `app/(a11y)/[locale]`.

## Active Locales

Configured in `lib/config/app-config.ts`:

- `en`
- `zh`
- `ja`

The default locale is `en`.

## URL Behavior

`middleware.ts` uses `localePrefix: "as-needed"` and `localeDetection: false`.

That means:

- English is unprefixed: `/`
- Chinese is prefixed: `/zh`
- Japanese is prefixed: `/ja`
- there is no browser-language redirect

## Configuration Files

### `lib/config/app-config.ts`

Central source of truth for:

- supported locales
- default locale
- locale labels
- localized site names
- CV data source settings

### `i18n.ts`

Exports:

- `locales`
- `defaultLocale`
- `localeLabels`
- `Locale`

It also provides the `next-intl` request config and loads `messages/{locale}.json`.

### `middleware.ts`

Applies locale-aware routing to all non-static, non-API paths and enables alternate links for SEO.

## Messages

Message files live in `messages/`:

- `messages/en.json`
- `messages/zh.json`
- `messages/ja.json`

There are also inactive files such as `messages/yue.json` and `messages/ko.json` retained for future expansion.

## Layout Integration

`app/(cv)/[locale]/layout.tsx`:

- validates the locale
- loads locale messages with `getMessages`
- passes `locale` and `messages` into `NextIntlClientProvider`
- sets `lang` and `dir` on `<html>`
- generates locale-aware metadata and alternate links

## Path Helpers

`lib/i18n-utils.ts` provides helpers used throughout the app:

- `isValidLocale`
- `getLocaleFromPathname`
- `createLocalizedPath`
- `getDirection`

`createLocalizedPath` preserves unprefixed default-locale URLs while prefixing non-default locales.

## Content Localization

### UI copy

Localized UI strings come from `messages/*.json`.

### CV content

Localized CV content comes from TOML:

- base source: `data/cv.toml`
- overrides: `data/cv.zh.toml`, `data/cv.ja.toml`

`lib/load-cv-data.ts` merges locale overrides on top of the base data by index.

### Secondary pages

Localized markdown content exists for:

- Accessibility statements
- Privacy statements
- generated `llms.txt`

## Adding A Locale

To enable a new locale:

1. Add the locale to `lib/config/app-config.ts`.
2. Add localized labels in `appConfig.intl.localeLabels`.
3. Create `messages/{locale}.json`.
4. Add `data/cv.{locale}.toml` if localized CV overrides are needed.
5. Add localized accessibility and privacy markdown files if needed.
6. Verify route generation, metadata, and footer controls.

If the locale is being staged but not enabled, its message and TOML files can remain in the repo without being added to the active locale list.

## Testing

Validate i18n changes with:

```bash
pnpm dev
pnpm check
```

Manual checks:

- `/`
- `/zh`
- `/ja`
- localized footer links
- localized `llms.txt`
- accessibility and privacy pages
